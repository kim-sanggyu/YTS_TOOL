# 공제요건 경계나이 관리 작업계획

## 1. 배경 및 목적

PAY_WRK_FMLY 데이터를 Y2025 → X2026으로 단순 마이그레이션 시, 연도가 바뀌면서 만나이가 경계값에 해당하는 부양가족의 공제요건이 달라질 수 있음.

- 예: 2025년에 만19세(기본공제 대상)였던 사람이 2026년에 만20세가 되어 기본공제 비대상이 되는 경우
- 이런 경우 주민번호를 조정하여 공제요건을 유지하도록 보정이 필요

**나이 경계값: 만 7, 20, 59, 69세**

---

## 2. 전체 플로우

```
[사전 준비 - 공제요건 경계나이 관리 화면]
  ① 경계나이 데이터 생성
     Y2025 기준 PAY_WRK_FMLY에서 경계나이 해당 레코드 검출
     → yttsDb.CALC_FMLY_AGE_DEFECT 에 INSERT
        ↓
  ② (선택) 담당자가 CHG_RES_NO(대체 주민번호) 직접 입력

[차년도 데이터 생성 화면]
  ③ C01 → C02 → C03(CALC_FMLY_AGE_DEFECT 참조) → C04 → ... → C15
     - CHG_RES_NO 있으면 → 대체 주민번호로 마이그레이션
     - CHG_RES_NO 없으면 → 원본 주민번호 그대로 마이그레이션
```

### 3-1. yttsDb 테이블 생성 ✅

```sql
CREATE TABLE CALC_FMLY_AGE_DEFECT (
  CALC_NO        VARCHAR2(20)   NOT NULL,
  YY             VARCHAR2(4),
  NM             VARCHAR2(100),             -- 본인명
  EMP_NO         VARCHAR2(10),
  KEEP_PS        VARCHAR2(1),
  FMLY_SEQ       NUMBER(3)      NOT NULL,
  FMLY_NM        VARCHAR2(100),             -- 부양가족명
  FMLY_RELN      VARCHAR2(21),              -- 관계코드
  RES_NO         VARCHAR2(500)  NOT NULL,   -- 원본 주민번호 (평문)
  MAN_AGE        NUMBER(3),                 -- 만나이
  BAS_SUB_YN     CHAR(1),                   -- 기본공제여부
  DEFECT_REASON  VARCHAR2(200),             -- 사유
  CHG_RES_NO     VARCHAR2(500),             -- 담당자 입력 대체 주민번호 (평문)
  STATUS         CHAR(1)  DEFAULT 'N',      -- N: 미처리 / Y: 처리완료
  INS_DT         DATE     DEFAULT SYSDATE,
  UPT_DT         DATE,
  CONSTRAINT PK_CALC_FMLY_AGE_DEFECT PRIMARY KEY (CALC_NO, FMLY_SEQ)
);
```

- DB: yttsDb (YTTS @ 61.111.37.10:11521/credb)
- RES_NO: 평문 저장 (이 작업 한정)

## 4. 남은 작업

### 4-1. 경계나이 데이터 생성 기능 (공제요건 경계나이 관리 화면)

**버튼 추가**: "경계나이 생성" 버튼

**실행 SQL** (ytsDb → yttsDb INSERT):

```sql
-- 먼저 기존 데이터 삭제 (재실행 대비)
DELETE FROM CALC_FMLY_AGE_DEFECT WHERE YY = :toYear;

-- Y2025 기준 경계나이 검출 후 INSERT
INSERT INTO CALC_FMLY_AGE_DEFECT (
  CALC_NO, YY, NM, EMP_NO, KEEP_PS,
  FMLY_SEQ, FMLY_NM, FMLY_RELN,
  RES_NO, MAN_AGE, BAS_SUB_YN, DEFECT_REASON, STATUS
)
WITH BASE AS (
  SELECT
    f.CALC_NO, f.FMLY_SEQ, f.RES_NO, f.NM AS FMLY_NM, f.FMLY_RELN, f.BAS_SUB_YN,
    m.NM, m.EMP_NO, w.KEEP_PS,
    :toYear - CASE SUBSTR(f.RES_NO, 7, 1)
                WHEN '1' THEN 1900 + TO_NUMBER(SUBSTR(f.RES_NO, 1, 2))
                WHEN '2' THEN 1900 + TO_NUMBER(SUBSTR(f.RES_NO, 1, 2))
                WHEN '3' THEN 2000 + TO_NUMBER(SUBSTR(f.RES_NO, 1, 2))
                WHEN '4' THEN 2000 + TO_NUMBER(SUBSTR(f.RES_NO, 1, 2))
                ELSE           1900 + TO_NUMBER(SUBSTR(f.RES_NO, 1, 2))
              END AS MAN_AGE
  FROM PAY_WRK_FMLY f
  JOIN PAY_WRK_FMLY m ON m.CALC_NO = f.CALC_NO AND m.FMLY_SEQ = 1
  JOIN PAY_WRK_MAIN w ON w.CALC_NO = f.CALC_NO
  WHERE f.CALC_NO LIKE '%Y' || :fromYear || '%'
    AND f.FMLY_SEQ <> 1
)
SELECT
  CALC_NO, :toYear, NM, EMP_NO, KEEP_PS,
  FMLY_SEQ, FMLY_NM, FMLY_RELN,
  RES_NO, MAN_AGE, BAS_SUB_YN,
  '만' || MAN_AGE || '세 (' || CASE BAS_SUB_YN WHEN 'Y' THEN '기본공제대상' ELSE '비대상' END || ')', 'N'
FROM BASE
WHERE MAN_AGE IN (7, 20, 59, 69);
```

**API**: `POST /api/tools/fmly-age-check/generate`
- ytsDb에서 조회 → yttsDb에 INSERT
- 파라미터: fromYear, toYear

### 4-2. CHG_RES_NO 입력 기능 (공제요건 경계나이 관리 화면)

- 테이블 행에서 CHG_RES_NO 셀을 클릭하여 직접 편집
- 저장 시 yttsDb UPDATE + UPT_DT 갱신

**API**: `PATCH /api/tools/fmly-age-check/update`
- 파라미터: CALC_NO, FMLY_SEQ, CHG_RES_NO

### 4-3. C03 마이그레이션에 보정 로직 반영

`src/features/data-migration/lib/scripts.ts` - c03 transformRow 수정:

```typescript
// C03 마이그레이션 시 CALC_FMLY_AGE_DEFECT 조회
// CHG_RES_NO가 있으면 RES_NO 교체
const defectMap = await loadDefectMap(toYear)  // yttsDb에서 조회

// transformRow 내부
const key = `${newRow.CALC_NO}_${row.FMLY_SEQ}`
if (defectMap[key]) {
  newRow.RES_NO = defectMap[key].CHG_RES_NO
}
```

- `run/route.ts`에서 C03 실행 전 defectMap 로드
- CHG_RES_NO 적용 건은 yttsDb STATUS → 'Y' 업데이트
