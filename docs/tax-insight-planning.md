# 연말정산 절세 분석 서비스 기획서

> 작성일: 2026-06-30

---

## 1. 배경 및 문제 인식

### 현재 상황
연말정산은 법적 의무 이행 중심으로 운영되어 왔다. 급여담당자의 목표는 "오류 없이 마감"이었고, 소득자에게 실질적 혜택을 주려는 노력은 없었다. 1년에 한 번 처리되는 업무 특성상 심도 있는 분석이 어려웠다.

### 발견된 기회
세액계산 Java 엔진이 계산 과정 전체를 텍스트로 기록(`CALC_PROC_TOTAL`)하고 있다. 원래 목적은 계산 검증이었으나, 이 데이터에는 절세 분석에 필요한 모든 재료가 담겨 있다.

**2025년 485명 데이터 분석 결과:**
| 항목 | 수치 | 의미 |
|------|------|------|
| 장애인전용보험 미가입 | 60명 중 57명 (95%) | 안내 한 번으로 1인당 최대 15만원 절세 |
| IRP·연금저축 미납입 | 433명 중 238명 (55%) | 1인당 최대 108만원 절세 가능 |
| 신용카드 비중 과다 | 292명 (67%) | 체크카드 전환으로 공제율 2배 |
| 고향사랑기부금 미활용 | 330명 (76%) | 실질 9,091원으로 90,909원 공제 |
| 의료비 최저한도 근접 | 68명 | 조금만 더 챙겼으면 공제 가능 |

## 2. 서비스 개요

### 핵심 가치
> "연말정산 데이터를 귀하게 다루는 새로운 시각"

세액계산 결과를 단순 신고 자료로 끝내지 않고, 소득자와 급여담당자 모두에게 실질적인 절세 인사이트를 제공한다.

### 서비스 구성

```
┌─────────────────────────────────────────┐
│           연말정산 절세 분석 서비스          │
├──────────────────┬──────────────────────┤
│  소득자용 서비스   │   급여담당자용 서비스   │
│  세액계산 종합진단  │   연말정산 대시보드    │
│                  │                      │
│  - 왜 이게 0원?   │  - 전체 현황 KPI      │
│  - 절세 기회      │  - 절세 기회 집계      │
│  - 잘 하고 있는것  │  - 총급여·세율 분포    │
└──────────────────┴──────────────────────┘
```

## 3. 핵심 기능

### 3-1. 세액계산 종합진단 (소득자·담당자 1인 조회)

**기능 ①: "왜 이게 0원이지?" — 공제 미적용 이유 설명**

소득자가 입력했는데 공제액이 0원이 된 이유를 자동으로 설명한다.

| 케이스 | 설명 예시 |
|--------|---------|
| 표준세액공제 → 월세 0원 | "표준세액공제 방식이 특별방식보다 XX만원 유리해서 자동 선택됐습니다. 표준방식에서는 월세 공제가 적용되지 않습니다." |
| 신용카드 최저사용금액 미달 | "총 XX원 사용하셨지만 최저사용금액 YY원(총급여×25%)에 ZZ원 부족합니다." |
| 연금저축+IRP 한도 초과 납입 | "합산 한도 900만원을 초과한 XX원은 세액공제 효과가 없습니다. 손실 공제액 약 YY원." |
| 실손보험 차감 후 의료비 없음 | "실손보험금 차감 결과 공제 대상 의료비가 없습니다." |
| 근로소득 조기 소진 | "국민연금 공제 단계에서 근로소득금액이 모두 소진됐습니다." |

**기능 ②: "절세할 거 없나?" — 절세 기회 진단**

세법 한도 기준과 본인 데이터를 비교해 미활용 항목을 찾아낸다.

- IRP·연금저축 추가 납입 여지 → 절감액 계산
- 장애인전용보험 미가입 (장애인 부양가족 있는 경우)
- 신용카드→체크카드 전환 전략
- 의료비 최저한도 근접 알림
- 고향사랑기부금 미활용
- "현재 최선" 판정 (해당 없으면 위안 메시지)

**화면 구성:**
```
[필터 ▼] ◄ [CALC_NO ▼] ► X / N

┌──────────────────┐ ┌──────────────────┐
│ CALC_PROC_TOTAL  │ │  기본 현황        │
│ (계산과정 원문)   │ │  ⚠️ 왜 0원이지?   │
│                  │ │  💡 절세 기회     │
│                  │ │  ✅ 잘 하고 있는것 │
└──────────────────┘ └──────────────────┘
```

### 3-2. 연말정산 대시보드 (급여담당자 전체 조회)

전체 신고자 집단을 분석해 급여담당자가 일괄 안내할 수 있는 인사이트를 제공한다.

- KPI 카드: 총 인원, 평균 총급여, 평균 실효세율, 환급/추가납부 현황
- 절세 기회 목록: 항목별 해당 인원·비율·설명
- 총급여 구간 분포, 실효세율 분포, 표준/특별 방식 비율

## 4. 데이터 구조

### 핵심 테이블: YTS39.PAY_WRK_CALC

| 컬럼 | 내용 | 활용 |
|------|------|------|
| `CALC_PROC_TOTAL` | 전체 소득·세액공제 계산 과정 (CLOB, 텍스트) | 왜 0원인지 설명, 성명·사번 추출 |
| `CALC_PROC_CARD` | 신용카드 상세 계산 (CLOB, JSON) | 카드 사용 구성 분석 |
| `CALC_PROC_MEDI` | 의료비 상세 계산 (CLOB, JSON) | 의료비 가족별 분석 |
| `CALC_PROC_INPUT` | **입력값 원본 JSON (추가 예정)** | 표준세액공제·소진 케이스 분석 |
| `RES_INCM_TAX` | 결정세액 (소득세) | 필터, 집계 |
| `SUB_INCM_TAX` | 차감징수세액 (음수=환급, 양수=추가납부) | 환급/납부 구분 |
| `CALC_METHOD` | 세액계산방식 기록 | 표준/특별 구분 |
| `EXHAUSTED_POINT` | 소득·세액 소진지점 | 조기소진 탐지 |

### CALC_PROC_INPUT 추가 계획 (마이그레이션)

```json
{
  "insu_grt":    800000,   // 보장성보험료
  "medi_total": 3500000,  // 의료비
  "edu_total":  2000000,  // 교육비
  "house_rent": 1200000,  // 월세
  "pension_pf": 3000000,  // 연금저축
  "pension_ret":5000000,  // IRP
  "don_relgn":  1000000   // 기부금 등
}
```
- RAW 입력 데이터 + 기존 Java 엔진으로 기존 데이터 백필 가능
- 추가 시 표준세액공제·소득 소진 케이스 분석 완전 해소

## 5. 기술 스택

### 현재 (YTS Tool 포털)
- **Frontend**: Next.js 16 (App Router) + TypeScript + Tailwind CSS + shadcn/ui
- **Backend**: Next.js API Routes (규칙 기반 분석 엔진)
- **DB**: Oracle 11g (oracledb v6.10.0)
- **분석 방식**: 규칙 기반 (AI 서버 불필요)

### 실 서비스 (Spring-Java-JSP)
- 분석 로직을 Java Service 클래스로 이식
- CALC_PROC_CARD/MEDI JSON → Jackson 파싱
- JSP 화면 구성

### 향후 검토 (AI 어시스턴트)
- Claude API 연동 → 급여담당자용 자연어 Q&A
- "장애인전용보험 미가입자 57명에게 보낼 안내문 써줘" 등
- 보안 정책 검토 후 결정 (개인 데이터 외부 전송 이슈)

- CALC_PROC_TOTAL 기반 설명 (Java 엔진 계산 결과를 그대로 해석)

## 7. 비즈니스 모델

### 수혜자 및 비용지불자 전환

```
초기 생각          현실
──────────         ──────────────────────
소득자 → 수혜자    인사팀 → 수혜자이자 비용지불자
소득자 → 지불자    (소득자는 수혜자지만 지불 주체 아님)
```

### 인사팀이 돈을 낼 수 있는 이유
- 집단 분석 결과를 들고 사장에게 직원 복리후생 개선 보고 가능
- 연말정산 민원 감소 → KPI 달성
- 기안이 아니라 성과 보고가 됨

### 현재 수준 수익화 방향
1. **기존 고객사**: 유지보수 계약에 분석 서비스 추가 항목으로 청구
2. **신규 고객사**: Java 세액계산 엔진 + 분석 서비스 패키지 판매
3. **레퍼런스 확보** → SaaS 전환 근거

### 장기 방향: SaaS
- 연말정산 시스템 자체를 구독 서비스로
- 엔진(Java) + 분석 + AI 어시스턴트 통합 패키지
- 진입장벽: 검증된 세액계산 엔진 + 누적 데이터
