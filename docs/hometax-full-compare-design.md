# 국세청 모의계산 — 전체비교 세분화 설계 & 미전송 백로그

> 작업 지도. 목표·설계·남은 일을 한 곳에. 진행하며 갱신한다.
> 관련: `src/features/hometax-calc/mapping/2025.ts`(단일 원천), `docs/hometax-yts-mapping-2025.md`(사람이 읽는 근거).

## 0. 최종 목표 (도장의 정의)

YTS 2026 세액계산 완료 데이터 **전체**를 국세청 모의계산에 부어 **산출세액·결정세액**을 대조한다.
불일치 = 내가 유지보수하는 세액계산 소스의 오류 후보 → 조사·수정.

**도장 = 차이가 0이 아니라 "설명 안 되는 차이"가 0.** 차이는 3종류로 분류한다:

| 종류 | 뜻 | 처리 |
|---|---|---|
| **A. 이해되는 차이** | 표준/특별 자동선택, 국세청에 입력구멍 없는 항목 | **수용(넘어감)** |
| **B. 아직 안 보낸 항목** | 미전송 send:false | **완전 전송으로 채움** (이 문서의 백로그) |
| **C. 다 보냈는데도 다름** | ← 진짜 사냥감 = 소스 버그 | **수정** |

도장 = B가 0이고 C가 전부 수정/이해된 상태.

## 1. 표 A — 단계별(흐름) 비교축 : "어느 단계에서 갈리나"

전체비교 탭을 이 흐름대로 세로 분해한다. (지금은 산출세액·결정세액 2줄만)
YTS 컬럼은 `PAY_WRK_CALC` 기준(reference-pay-wrk-calc-columns) — 빌드 시 실제 컬럼 재확인.

| # | 단계 | YTS 컬럼 | NTS 결과코드 |
|---|---|---|---|
| 1 | 총급여 | `TOT_PAY_AMT` | 8900(입력) |
| 2 | 근로소득공제 | `WORK_TAX` | 8901 |
| 3 | 근로소득금액 | `WORK_AMT` | 8902 |
| 4 | **소득공제 합** | `WORK_AMT − TOT_PTB` | `8902 − 8903` |
| 5 | 종합소득 과세표준 | `TOT_PTB` | 8903 |
| 6 | **산출세액** | `PROD_TAX_AMT` | 8990 |
| 7 | 근로소득세액공제 | `RT_WIA` | 8923 |
| 8 | **세액공제 합** | `RT_SUM` | `8990 − 8999`(−감면) |
| 9 | **결정세액** | `RES_INCM_TAX` | 8999 |
| 10 | 지방소득세 | `RES_INHABT_TAX` | 8998 |
| 11 | 차감징수 | `SUB_INCM_TAX` | 8992 |

**진단 원리** — 위→아래 처음 갈리는 지점이 원인 band:
- 근로소득금액(3) 일치 + 과세표준(5) 불일치 → **소득공제 band** → 표 B 소득공제 드릴
- 과세표준(5) 일치 + 산출세액(6) 불일치 → 세율표 문제(거의 없음)
- 산출세액(6) 일치 + 결정세액(9) 불일치 → **세액공제/감면 band** → 표 B 세액공제 드릴

관련 참고: `2025.ts` `NTS_RESULT_CODES`. 소득공제 계 `SPCL_SUB_AMT_SUM`+`OTO_SUM`, 세액공제 계 `RT_SUM`, 세액감면 계 `TAX_CUT`, 표준세액공제 `RT_BASE_SUB_AMT`.

## 4. 진행 레시피 (그룹 1개 = 이 순서)

완료 4그룹에서 확립된 패턴 + 이제 테스트 잠금 추가:

1. **프로브 실측** — `docs/hometax-{group}-probe.mjs` 로 국세청에 시험 전송, 몇 명 **원단위 일치** 확인 → send→receive 계약 확정.
2. **매핑 확정** — `mapping/{group}.ts`(필요시) + `2025.ts` 해당 행 `send: true` flip.
3. **주입 배선** — `route.ts`/`runCompareForCalcNo.ts` inject{Group}Vals + 가상컬럼 SQL 제외.
4. **표시** — `list/route.ts` type 분기 + `HometaxCalcPanel` 탭(있으면) + 전체비교 표 A/B 반영.
5. **응답 보장** — `runHometaxCalc.ts` ALL_CODES에 결과코드 추가.
6. **★골든 테스트로 잠금** — 순수 매핑/파싱 함수는 `giftNtsCode`처럼 vitest 테스트. ([[project-harness-setup]])
7. tsc+eslint 클린 → 라이브 확인 → 표 B 갱신(❌→✅).
