### 핵심 결과물
- **실행과정 드로어 재구성**: ①결과비교 ②전송한 공제입력 ③NTS원본IN/OUT 3개 접이식 패널(`DetailPanel`). 단일클릭=개별토글, **더블클릭=그 패널만 최대화(`flex-1`)하고 나머지는 헤더만 접힘**(`focusedPanel` 상태). 각 패널 독립 세로/가로 스크롤 + `thead sticky`로 헤더 고정. 좌(계산과정)-우(실행과정) 드로어 자체도 드래그 리사이즈 가능. 두 드로어(계산과정 단독/실행+계산 나란히) 모두 우상단에 전체보기 토글(사이드바 아이콘폭 3rem만 남기고 나머지 전체 차지).
- **①결과비교 범위 축소**: 총급여(8900)→결정세액(8999)까지만. **기납부세액(8991)·차감징수(8992)·지방소득세(8998) 관련 코드 전량 제거** — 레거시 `runHometaxCalc()`(총급여+기납부 팝업 방식) 함수, `route.ts`의 `mode:"simple"` 분기까지 통째로 삭제.
- **버그 정정(실측 확인)**: 근로소득세액공제의 NTS 코드가 **8923이 아니라 8700**이었음. 총급여 5천만원 단독 입력 라이브 프로브로 확인 — 8923은 "세액공제 합계"(8700+8770 표준세액공제 등)였고, 개별 근로소득세액공제 결과는 8700에 옴. `mapping/2025.ts`(NTS_RESULT_CODES) · `HometaxCalcPanel.tsx`(NTS_FLOW, PROC_LABEL_CODE, compareRows) 전부 수정.
- **소계형 outCode 배선**: 카드(8431~8463→8430), 의료비(8720/8721/8725/8729→8726)에 `outCode` 추가. 카드는 `resultCol:"OTO_CARD_ETC"`도 추가해 YTS쪽 소계 대조 가능해짐. ②패널에 그룹 마지막에 "카드 소계"/"의료비 소계" 행 별도 삽입.
- **②전송한 공제입력 단순화**: YTS컬럼·전송값(가공)·NTS반영(가공) 제거 → **NTS IN/OUT 원본값 그대로**(항목/NTS코드/보낸값/받은값 4열). "전체보기/값보기" 토글 추가(값보기 = 화면에 실제 보이는 보낸값·받은값 중 하나라도 있는 행만). YTS 대조는 ③패널 전담으로 분리.
- **드로어 데이터 조회 크로스탭 버그 수정**: 상세조회 드로어의 `detailRow`가 `allItems`(전체탭)에서만 찾던 걸, 기부금/카드/의료비/연금/기타 탭 리스트까지 다 뒤지도록 확장(`DetailRowLike`). 전체탭 안 들어가고 다른 탭에서 바로 열어도 계산과정이 정상 표시됨.
